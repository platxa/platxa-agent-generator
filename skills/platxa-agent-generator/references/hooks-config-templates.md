# Hook Configuration Templates

Reference templates for Claude Code hook configurations generated by the Platxa Agent Generator.
Use these templates when generating `.claude/settings.json` hook blocks via the Write tool.

## Hook Event Types

| Event | Description | Matcher Support |
|-------|-------------|----------------|
| `SessionStart` | Session begins | No |
| `Stop` | Session ends / stop hook | No |
| `PreToolUse` | Before tool invoked | Yes (`tool_name`) |
| `PostToolUse` | After tool completes | Yes (`tool_name`) |
| `Notification` | Notifications sent | No |
| `SubagentStart` | Subagent spawned | No |
| `SubagentStop` | Subagent completes | No |
| `PreCompact` | Before context compaction | No |
| `UserPromptSubmit` | User submits prompt | No |
| `TeammateIdle` | Agent team member goes idle | No |
| `TaskCreated` | Task created via TaskCreate | No |
| `TaskCompleted` | Task marked complete | No |

## Tool Matchers

| Matcher Key | Pattern | Use Case |
|-------------|---------|----------|
| `write` | `Write\|Edit\|MultiEdit` | File modification operations |
| `read` | `Read\|Glob\|Grep` | File reading/search operations |
| `bash` | `Bash` | Shell execution |
| `task` | `Task` | Subagent spawning |
| `all_file_ops` | `Write\|Edit\|MultiEdit\|Read` | All file access |
| `dangerous` | `Write\|Edit\|MultiEdit\|Bash` | File-modifying + shell ops |

## settings.json Hook Block Structure

```json
{
  "hooks": {
    "EventName": [
      {
        "matcher": "tool_name_pattern",
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/script.sh"
          }
        ]
      }
    ]
  }
}
```

For events without matcher support, omit the `matcher` field entirely.

| Type | Events | Matcher | Behavior |
|------|--------|---------|----------|
| audit | PreToolUse, PostToolUse, SessionStart, Stop | varies | JSONL logging to file |

## Dangerous Command Patterns

Regex patterns for PreToolUse deny hooks (aligned with security_scanner.py):

| Code | Pattern | Description |
|------|---------|-------------|
| SEC001 | `rm\s+-rf\s+[/~]` | Destructive file deletion |
| SEC002 | `\bsudo\b` | Privileged execution |
| SEC003 | `chmod\s+777\b` | Insecure file permissions |
| SEC004 | `\beval\s*\(` | Dynamic code execution (eval) |
| SEC005 | `\bexec\s*\(` | Dynamic code execution (exec) |
| SEC006 | `>\s*/dev/sd[a-z]` | Direct disk write |
| SEC007 | `dd\s+.*of=/dev/` | Direct disk write with dd |
| SEC008 | `:\s*\(\)\s*\{` | Fork bomb pattern |

Also blocked by command string matching:

| Pattern | Reason |
|---------|--------|
| `; rm ` | Command injection with rm |
| `\| rm ` | Pipe to rm |
| `&& rm -rf` | Chained destructive deletion |
| `$(curl` | Remote code execution via curl |
| `` `curl `` | Remote code execution via curl |
| `\| sh` | Pipe to shell |
| `\| bash` | Pipe to bash |

## Completion-Promise Stop Hook

The canonical completion-promise check script verifies that the agent has
emitted `<promise>COMPLETE</promise>` before terminating.

**settings.json block:**
```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/.claude/hooks/team-lead-completion-check.sh"
          }
        ]
      }
    ]
  }
}
```

**Script template** (`team-lead-completion-check.sh`):
```bash
#!/usr/bin/env bash
# Completion-promise marker check for team-lead agent.
# Reads JSON payload from stdin; exits 0 (allow stop) or 2 (block stop).
set -euo pipefail

INPUT=$(cat)
STOP_HOOK_ACTIVE=$(echo "$INPUT" | python3 -c "
import json,sys
d=json.load(sys.stdin)
print('true' if d.get('stop_hook_active') else 'false')
" 2>/dev/null || echo "false")

if [ "$STOP_HOOK_ACTIVE" = "true" ]; then
    exit 0
fi

MAX_ITERATIONS=${MAX_ITERATIONS:-5}
CURRENT_ITERATION=$(echo "$INPUT" | python3 -c "
import json,sys
print(json.load(sys.stdin).get('current_iteration', 0))
" 2>/dev/null || echo "0")

if [ "$CURRENT_ITERATION" -ge "$MAX_ITERATIONS" ] 2>/dev/null; then
    echo "Iteration cap reached ($CURRENT_ITERATION/$MAX_ITERATIONS) — allowing termination." >&2
    exit 0
fi

MESSAGE=$(echo "$INPUT" | python3 -c "
import json,sys
print(json.load(sys.stdin).get('last_assistant_message', ''))
" 2>/dev/null || echo "")

MARKER="<promise>COMPLETE</promise>"
if echo "$MESSAGE" | grep -qF "$MARKER"; then
    exit 0
fi

echo "Completion marker not found in assistant output." >&2
exit 2
```

## PreToolUse Deny Hook

Blocks dangerous commands before invocation.

**Script template** (`pretooluse-deny.sh`):
```bash
#!/usr/bin/env bash
set -euo pipefail
INPUT=$(cat)
TOOL=$(echo "$INPUT" | python3 -c "import json,sys; print(json.load(sys.stdin).get('tool_name',''))" 2>/dev/null || echo "")
DENIED_TOOLS="${DENIED_TOOLS:-}"
for denied in $DENIED_TOOLS; do
    if [ "$TOOL" = "$denied" ]; then
        echo "Tool '$TOOL' is not permitted for this agent." >&2
        exit 2
    fi
done
exit 0
```

**settings.json block:**
```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": ".*",
        "hooks": [
          {
            "type": "command",
            "command": "DENIED_TOOLS='Bash WebSearch' /path/to/pretooluse-deny.sh"
          }
        ]
      }
    ]
  }
}
```

## PostToolUse Lint Hook

Runs linting after Write/Edit tool calls. Supported linters:

| Extension | Linter | Command |
|-----------|--------|---------|
| `.py` | ruff | `ruff check --no-fix` |
| `.ts`, `.tsx`, `.js`, `.jsx` | eslint | `eslint --no-fix` |
| `.go` | golangci-lint / go vet | `golangci-lint run` |
| `.rs` | cargo clippy | `cargo clippy --message-format=short` |

**settings.json block:**
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit|MultiEdit",
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/{agent-name}-auto-lint.sh"
          }
        ]
      }
    ]
  }
}
```

## Stop Verification Gate

Blocks agent completion until the test suite passes. Auto-detects framework:

| Detection | Framework | Command |
|-----------|-----------|---------|
| `pyproject.toml`, `pytest.ini`, `conftest.py` | pytest | `pytest --tb=short -q` |
| `vitest.config.ts` | vitest | `vitest run --reporter=verbose` |
| `go.mod` | go test | `go test ./...` |

## Hook Validation Rules

| Check | Rule |
|-------|------|
| Agent name | Must match `^[a-zA-Z0-9_-]+$` (no shell metacharacters, no path traversal) |
| Script paths | Must be absolute or relative to project root |
| Event names | Must be from the supported event table above |
| Matcher fields | Only for events that support matchers (PreToolUse, PostToolUse) |
| Exit codes | `0` = allow/continue, `2` = block/deny, `1` = error |

## Domain-Specific Hook Recommendations

| Agent Domain | Recommended Hooks |
|-------------|-------------------|
| Security scanner | PreToolUse deny (Bash), PostToolUse lint |
| Orchestrator | Stop (completion-promise), TeammateIdle |
| Code reviewer | SessionStart (context load), PostToolUse lint |
| Test generator | PostToolUse lint, Stop (observation) |
| Team lead | Stop (completion-promise + iteration cap) |
