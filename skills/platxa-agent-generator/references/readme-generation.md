# README Generation Reference

Templates for generating README.md agent catalogues. The generation-subagent
uses Glob to discover agents, Read to extract metadata, and Write to render
the final README.

## README Structure

```markdown
# {title}

{overview paragraph}

## Agent Catalogue

| Name | Description | Tools |
|------|-------------|-------|
| agent-name | What it does | Read, Grep, ... |

## Agent Details

### agent-name

**Description:** What the agent does.

**Tools:** Read, Grep, Glob

**Usage Example:**

```
Task(subagent_type="agent-name",
     description="...",
     prompt="...")
```
```

## Agent Summary Fields

Each agent entry requires these fields extracted from frontmatter + body:

| Field | Source | Fallback |
|-------|--------|----------|
| name | `name:` in frontmatter | filename without `.md` |
| description | `description:` in frontmatter | first paragraph of body |
| tools | `tools:` in frontmatter | empty list |
| usage_example | `## Examples` section body | synthesized Task() snippet |
| source_path | file path relative to agents/ | — |

## Example Extraction

Look for example sections using these heading patterns (in priority order):

1. `## Usage Example`
2. `## Example`
3. `## Examples`
4. `### Example`

If no example section exists, synthesize a Task tool snippet:

```
Task(subagent_type="{agent_name}",
     description="Brief description of the task",
     prompt="Your request here")
```

## Table Cell Escaping

When rendering markdown tables:

- Escape pipe characters: `|` → `\|`
- Collapse newlines to spaces
- Truncate descriptions longer than 80 characters with `...`

## Output Conventions

- **Filename:** `README.md`
- **Location:** same directory as the agents it documents
- **Encoding:** UTF-8, LF line endings
- **Footer:** `*Generated by Platxa Agent Generator*`
