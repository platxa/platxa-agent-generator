# 60-30-10 Color Rule Enforcement

The 60-30-10 rule is a classic interior design principle adapted for UI design. It creates visual harmony by distributing colors in specific proportions across the interface.

## The Rule

| Proportion | Role | Purpose | Tokens |
|------------|------|---------|--------|
| **60%** | Dominant | Background, large areas | `background`, `card`, `muted` |
| **30%** | Secondary | Supporting elements | `secondary`, `border`, `input` |
| **10%** | Accent | Call-to-action, highlights | `primary`, `accent`, `destructive` |

## Token Mapping

### 60% Dominant Colors

Used for large surface areas and backgrounds:

```css
/* Primary surfaces */
--background: oklch(0.98 0 0);      /* Page background */
--card: oklch(0.99 0 0);            /* Card surfaces */
--muted: oklch(0.95 0.01 250);      /* Muted sections */

/* In components */
.page { @apply bg-background; }
.card { @apply bg-card; }
.section { @apply bg-muted; }
```

**Usage in Components:**
```typescript
// Page layouts - 60%
<main className="min-h-screen bg-background">
  <div className="bg-card rounded-lg">
    {/* Card content */}
  </div>
</main>

// Large sections - 60%
<section className="bg-muted py-16">
  {/* Section content */}
</section>
```

### 30% Secondary Colors

Used for supporting UI elements:

```css
/* Supporting elements */
--secondary: oklch(0.95 0.02 250);        /* Secondary buttons */
--secondary-foreground: oklch(0.15 0.02 250);
--border: oklch(0.90 0.01 250);           /* Borders */
--input: oklch(0.90 0.01 250);            /* Input backgrounds */
--muted-foreground: oklch(0.45 0.02 250); /* Subdued text */

/* In components */
.border { @apply border-border; }
.input { @apply border-input; }
.secondary-text { @apply text-muted-foreground; }
```

**Usage in Components:**
```typescript
// Borders and dividers - 30%
<div className="border-b border-border">
<hr className="border-border" />

// Secondary buttons - 30%
<Button variant="secondary">Cancel</Button>
<Button variant="outline">Settings</Button>

// Input fields - 30%
<Input className="border-input bg-background" />

// Supporting text - 30%
<p className="text-muted-foreground text-sm">
  Helper text goes here
</p>
```

### 10% Accent Colors

Used sparingly for important actions and highlights:

```css
/* Accent elements */
--primary: oklch(0.6 0.2 250);            /* Primary actions */
--primary-foreground: oklch(0.98 0 0);
--accent: oklch(0.95 0.02 250);           /* Hover states */
--accent-foreground: oklch(0.15 0.02 250);
--destructive: oklch(0.55 0.22 25);       /* Danger actions */
--ring: oklch(0.6 0.2 250);               /* Focus rings */

/* In components */
.primary-button { @apply bg-primary text-primary-foreground; }
.link { @apply text-primary; }
.focus { @apply ring-ring; }
```

**Usage in Components:**
```typescript
// Primary CTAs - 10%
<Button variant="default">Submit</Button>
<Button variant="destructive">Delete</Button>

// Links - 10%
<a className="text-primary hover:underline">Learn more</a>

// Focus indicators - 10%
<button className="focus-visible:ring-2 focus-visible:ring-ring">

// Status indicators - 10%
<Badge variant="default">New</Badge>
<span className="text-destructive">Error message</span>
```

## Enforcement Rules

### Design Analyzer Checks

The design-analyzer validates color distribution:

```typescript
interface ColorAnalysis {
  dominant: number    // Should be ~60%
  secondary: number   // Should be ~30%
  accent: number      // Should be ~10%
  violations: ColorViolation[]
}

interface ColorViolation {
  element: string
  issue: string
  suggestion: string
}

function analyzeColorBalance(component: string): ColorAnalysis {
  const colorUsage = extractColorUsage(component)

  const violations: ColorViolation[] = []

  // Check for accent overuse
  if (colorUsage.accent > 0.20) {
    violations.push({
      element: "component",
      issue: "Accent colors exceed 20% usage",
      suggestion: "Replace some bg-primary with bg-card or bg-muted"
    })
  }

  // Check for missing dominant
  if (colorUsage.dominant < 0.50) {
    violations.push({
      element: "layout",
      issue: "Not enough neutral background space",
      suggestion: "Add bg-background or bg-card to container"
    })
  }

  // Check for CTAs without accent
  if (hasCallToAction(component) && colorUsage.accent < 0.05) {
    violations.push({
      element: "CTA",
      issue: "Call-to-action lacks visual prominence",
      suggestion: "Use bg-primary or text-primary for CTAs"
    })
  }

  return {
    dominant: colorUsage.dominant,
    secondary: colorUsage.secondary,
    accent: colorUsage.accent,
    violations
  }
}
```

### Token Classification

```typescript
const colorTokens = {
  dominant: [
    "background",
    "card",
    "popover",
    "muted"
  ],
  secondary: [
    "secondary",
    "border",
    "input",
    "muted-foreground",
    "secondary-foreground"
  ],
  accent: [
    "primary",
    "primary-foreground",
    "accent",
    "accent-foreground",
    "destructive",
    "destructive-foreground",
    "ring"
  ]
}

function classifyColor(className: string): "dominant" | "secondary" | "accent" | "neutral" {
  for (const [category, tokens] of Object.entries(colorTokens)) {
    for (const token of tokens) {
      if (className.includes(token)) {
        return category as "dominant" | "secondary" | "accent"
      }
    }
  }
  return "neutral"
}
```

## Anti-Patterns

### ❌ Too Much Accent

```typescript
// Wrong: Primary color dominates
<div className="bg-primary min-h-screen">
  <Card className="bg-primary-foreground">
    <Button>Action</Button>
  </Card>
</div>
```

### ❌ No Visual Hierarchy

```typescript
// Wrong: Everything same weight
<div className="bg-gray-100">
  <Card className="bg-gray-200">
    <h2 className="text-gray-700">Title</h2>
    <p className="text-gray-700">Content</p>
    <Button variant="secondary">Action</Button>
  </Card>
</div>
```

### ❌ Competing Accents

```typescript
// Wrong: Multiple accent colors fighting
<Card>
  <Button variant="default">Primary</Button>
  <Button variant="destructive">Danger</Button>
  <Badge className="bg-green-500">Success</Badge>
  <Badge className="bg-yellow-500">Warning</Badge>
</Card>
```

## Best Practices

| Do | Don't |
|----|-------|
| Use neutral backgrounds for large areas | Make entire sections primary-colored |
| Reserve accent for 1-2 elements per view | Use accent on every interactive element |
| Create hierarchy through color weight | Use all colors at equal visual weight |
| Use muted-foreground for supporting text | Make all text high contrast |
| Single primary CTA per section | Multiple competing primary buttons |
| Subtle borders with border-border | Bold colored borders everywhere |
